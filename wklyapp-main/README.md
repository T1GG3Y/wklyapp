# WKLY - Weekly Budget Tracker

A Next.js app for tracking weekly finances, now with Capacitor support for Android.

## Getting Started

### Prerequisites

- Node.js 18+ and npm
- Android Studio (for Android development)
- Java JDK 11 or higher

### Installation

1. Install dependencies:
```bash
npm install
```

2. Set up Capacitor (first time only):
```bash
# Make scripts executable (macOS/Linux)
chmod +x scripts/*.sh

# Run setup script
./scripts/setup-capacitor.sh
```

Or manually:
```bash
# Initialize Capacitor
npx cap init "WKLY" "com.tiggey.wkly" --web-dir=out

# Add Android platform
npx cap add android

# Copy google-services.json
cp /path/to/google-services.json android/app/google-services.json

# Build and sync
npm run build
npx cap sync android
```

## Development

### Web Development
```bash
npm run dev
```

### Android Development

1. Build the Next.js app:
```bash
npm run build
```

2. Sync with Capacitor:
```bash
npm run cap:sync
```

3. Open in Android Studio:
```bash
npm run cap:open
```

4. Run on device/emulator from Android Studio or:
```bash
npm run cap:run
```

## Building for Production

### Android APK/AAB

1. Build the web app:
```bash
npm run build
```

2. Sync Capacitor:
```bash
npm run cap:sync
```

3. Open Android Studio:
```bash
npm run cap:open
```

4. In Android Studio:
   - Configure signing in `android/app/build.gradle` (see below)
   - Build > Generate Signed Bundle / APK
   - Select "Android App Bundle" for Play Store or "APK" for direct distribution

### Android Signing Configuration

Edit `android/app/build.gradle` and add signing config:

```gradle
android {
    ...
    signingConfigs {
        release {
            storeFile file('path/to/your/keystore.jks')
            storePassword 'your-store-password'
            keyAlias 'your-key-alias'
            keyPassword 'your-key-password'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            ...
        }
    }
}
```

**Important**: Never commit your keystore file or passwords to version control!

## Firebase Configuration

The app uses Firebase for authentication and data storage. The `google-services.json` file must be placed in `android/app/` directory for Android builds.

## Available Scripts

- `npm run dev` - Start development server
- `npm run build` - Build for production
- `npm run build:android` - Build and sync for Android
- `npm run cap:sync` - Sync web assets to native projects
- `npm run cap:open` - Open Android project in Android Studio
- `npm run cap:copy` - Copy web assets to native projects
- `npm run cap:run` - Run on connected device/emulator

## Troubleshooting

### Build Issues

If you encounter build errors:
1. Clean the build: `rm -rf out android/.gradle`
2. Rebuild: `npm run build && npm run cap:sync`

### Firebase Issues

Ensure `google-services.json` is in `android/app/` and that the package name matches `com.tiggey.wkly`.

### Capacitor Sync Issues

If assets aren't updating:
1. Delete `android/app/src/main/assets/public`
2. Run `npm run cap:sync`
